1. Project Overview

We are combining two recent multi-agent reinforcement learning (MARL) methods:

  • ExpoComm (Li et al., ICLR 2025) — learns who to communicate with using an exponential graph topology (O(log N) edges per agent).
  • BVME (AAMAS 2026) — learns how much information to send by compressing messages through a variational bottleneck.

Neither paper addresses both problems simultaneously. Our contribution, ExpoComm-B, integrates BVME’s bandwidth-constrained variational encoding directly into ExpoComm’s sparse communication framework. This is the first evaluation of these two techniques combined.

2. Progress Summary

Implementation (Complete)

We implemented the BVME module from scratch (no public code exists) and integrated it into the ExpoComm codebase. The key components we built are:

  • BVMEModule — a variational encoder that maps messages to Gaussian distributions (μ, σ), samples compressed representations using the reparameterization trick, and computes KL divergence loss against a tunable prior.
  • ExpoComm-B Agent — extends the ExpoComm one-peer agent by inserting the BVME bottleneck between the message aggregation step and the Q-network.
  • BVME Q-Learner — a custom training loop that optimizes the combined loss: TD error + auxiliary state prediction loss + KL divergence penalty.

All components were unit tested and registered into the EPyMARL framework’s configuration system.

Infrastructure (Complete)

  • Deployed on the Rice NOTS HPC cluster using Miniforge + Python 3.8 + PyTorch 1.13.1.
  • Configured Weights & Biases (W&B) for remote experiment tracking and visualization.
  • Built parameterized SLURM scripts that use $SHARED_SCRATCH for job I/O in compliance with cluster policies.
  • Code is version-controlled at github.com/vt20-crypto/ExpoComm-B.

Environment Selection

We are currently using MAgent Battle (PettingZoo) as our primary benchmark — a large-scale grid-world battle with ~81 agents per team. We chose this over SMACv2 (StarCraft II) because the cluster’s 10 GB home directory quota cannot accommodate the 3.5 GB StarCraft II installation. SMACv2 experiments are planned for AWS if time permits.

3. Preliminary Results

Our first full training run of ExpoComm-B completed on the NOTS GPU cluster (24 hours, 3.35M timesteps on an NVIDIA GPU node).

The agents learned to consistently defeat the pretrained opponent team while communicating through the BVME variational bottleneck. The stable KL loss confirms that the compression mechanism is actively constraining information flow as intended.
